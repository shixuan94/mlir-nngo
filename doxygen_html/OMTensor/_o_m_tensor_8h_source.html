38 /* Typically, MemRefs in MLIR context are used as a compile-time constructs.
39  * Information such as element type and rank of the data payload is statically
40  * encoded, meaning that they are determined and fixed at compile-time. This
41  * presents significant burden for any runtime components trying to interact
42  * with the compiled executable.
43  *
44  * Thus a version of MemRef struct that is amenable to runtime manipulation is
45  * provided as a basis for building any runtime-related components providing
46  * user-facing programming interfaces. All information are dynamically encoded
47  * as members of this struct so that they can be accessed and modified easily
48  * during runtime.
49  *
50  * We will refer to it as an OMTensor.
51  */
